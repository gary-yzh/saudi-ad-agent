# ADR 000: Template

> Copy this file for new decisions. Number sequentially (001, 002, ...).
> Keep ADRs short — 100-300 words is the sweet spot. If you need more,
> the decision probably needs more thinking first.

## Status

`Proposed` → `Accepted` → `Superseded by ADR-XXX` / `Deprecated`

(One of these four. Date the status change in the body if it changes.)

## Context

What's the problem we're solving? What constraints exist? What forces
are pulling in different directions?

Be concrete. "Doubao API rejected duration=23 with HTTP 400" beats
"we had video generation issues".

## Decision

What did we pick? One clear sentence is enough; bullets for the
specifics.

## Consequences

What gets easier, what gets harder, what did we give up.

* **Gain:** ...
* **Cost:** ...
* **Alternatives considered:** A (rejected because X), B (rejected
  because Y).

If the decision goes wrong later, this section is where the trace
of your reasoning lives. Future you will thank present you.
